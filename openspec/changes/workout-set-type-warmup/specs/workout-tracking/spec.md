## ADDED Requirements

### Requirement: 组类型（正式组/热身组）枚举与录入

每一组（`WorkoutSet`）SHALL 带一个**组类型**，由可扩展枚举承载，当前取值为 `working`（正式组）与 `warmup`（热身组），默认 `working`。新建组、历史既有组与任何未识别的类型值 MUST 视为正式组（`working`），以保证旧数据与跨版本兼容。

录入时，组行最左的序号徽章 SHALL 作为类型切换入口：用户点击徽章在 `正式组 ⇄ 热身组` 间切换；同时组级「更多操作」⋯ 菜单 SHALL 提供「标为热身组 / 改回正式组」入口（与徽章点按等价，提升可发现性）。徽章显示规则——热身组 MUST 显「热」（朱砂红浅底 pill）且不参与数字编号；正式组 MUST 按同一动作内**仅正式组**的相对顺序重新编号为 `1, 2, 3…`。同一动作下，热身组 SHALL 强制排在所有正式组之前（自动吸顶），正式组之间保持用户手动顺序。「加一组」MUST 默认追加正式组，并以上一**正式**组重量作为预填源。

#### Scenario: 默认与历史均为正式组
- **WHEN** 用户新增一组，或读取本 change 之前记录的历史组
- **THEN** 该组类型为正式组（`working`），徽章显数字、计入统计

#### Scenario: 徽章点击切换类型
- **WHEN** 用户点击某正式组的序号徽章
- **THEN** 该组变为热身组，徽章显「热」、自动排到该动作热身段、其余正式组重新编号 1..n；再次点击该「热」徽章（或 ⋯ 菜单「改回正式组」）则切回正式组

#### Scenario: 仅正式组参与编号
- **WHEN** 某动作含 2 个热身组与 3 个正式组
- **THEN** 两个热身组徽章均显「热」并排在最前，三个正式组依次显 1 / 2 / 3

#### Scenario: 加一组默认正式组
- **WHEN** 焦点在某动作并按「加一组」
- **THEN** 末尾追加一个**正式组**（预填上一正式组重量），而非热身组

#### Scenario: 未识别类型兜底
- **WHEN** 客户端读到本端未识别的组类型值（如来自更高版本的扩展类型）
- **THEN** 按正式组处理，不崩溃，且计入统计

### Requirement: 热身组排除于统计与 PR

所有训练统计 SHALL 以「正式组 = 组类型 `!= warmup`」为唯一判据，热身组 MUST NOT 计入。受此约束的口径包括：PR 最大重量（动作库行与动作详情）、周训练量（volume）、周总组数、周总次数、历史强度曲线。热身组记录本身 MUST 完整保留（重量/次数照常录入与展示），仅在统计聚合时被排除。

判据 SHALL 表达为「排除 `warmup`」而非「仅取 `working`」，使将来新增的正式类组类型自动计入，无需改动统计逻辑。

#### Scenario: 热身组不计训练量与组数
- **WHEN** 某动作有 2 个热身组（各 40kg×10、60kg×5）与 3 个正式组（各 80kg×8）
- **THEN** 周训练量只累加 3 个正式组、周总组数只计 3、热身组的 reps 不计入总次数

#### Scenario: 热身组不破 PR
- **WHEN** 某动作的最大重量出现在一个被标为热身组的组里
- **THEN** PR 与历史曲线忽略该热身组，仅按正式组计算

#### Scenario: 热身组记录仍保留
- **WHEN** 用户把某组标为热身组
- **THEN** 该组的重量/次数仍在训练详情中展示，不被删除，仅不进统计

## MODIFIED Requirements

### Requirement: 训练首页周聚合视图

iOS 训练首页 SHALL 在顶部展示「本周训练量」hero（按本地时区周一 00:00 为周起点），并以两宫格展示本周「总组数 / 总次数」。聚合数据 MUST 按需即时计算自本地 `Workout` 集合，不入库。App MUST NOT 展示或计算「平均时长」统计。**训练量、总组数、总次数 MUST 仅统计正式组（组类型 `!= warmup`），热身组不计入。** 当本周训练数为 0 时，hero MUST NOT 显示「0.0 t」字面量，而 SHALL 展示鼓励性 Empty State 文案与「开始第 1 次训练」CTA。

#### Scenario: 本周已有训练
- **WHEN** 用户进入训练首页，本周已记录 4 次训练，正式组总训练量 28.4 吨
- **THEN** Hero 显示「28.4 t」并附副标「已完成 4 / Y 次」（Y 为当前激活计划的周训练目标次数，无激活计划时改为「本周第 4 次训练」）

#### Scenario: 本周尚未训练
- **WHEN** 本周训练数为 0
- **THEN** Hero 显示「准备好了吗？」文案与「开始第 1 次训练」CTA，不显示「0.0 t」

#### Scenario: 两宫格缺值
- **WHEN** 某一项聚合为 0
- **THEN** 显示「—」而非「0」

#### Scenario: 热身组不计入周聚合
- **WHEN** 本周训练里含若干热身组
- **THEN** 总组数/总次数/训练量均不包含这些热身组，仅按正式组聚合
