# Proposal — 历史训练日历

## Why

历史训练展示曾在 `remove-history-poster-profile-pages` 中整体移除，训练数据仍保留但缺少可视化入口。用户导入大量历史训练后，需要能按日期快速查看训练节奏与当天记录，同时不能把首页重新拖回全量历史扫描路径。

## What

- 在训练首页恢复轻量历史入口，进入独立历史日历页。
- 以月历呈现训练日；有训练的日期直接展示训练量与训练标题摘要，点选日期后用底部抽屉展示当天训练列表。
- 支持点击独立月份选择按钮打开月份档案列表，按年份扫视每月训练密度并直接跳转指定月份。
- 将「最长连续」从历史日历和个人中心等 App 模块中移除。
- 复用 `WorkoutDetailView` 查看单次训练详情。
- 扩展 `WorkoutHistoryStore` 的内存 projection，日历页只读取纯值摘要，不直接订阅完整训练聚合树。
- 不新增后端 API；服务端仍只承担既有同步能力。

## Impact

- iOS：新增历史日历 UI 与 calendar projection。
- OpenSpec：恢复并重新定义训练历史日历需求，保留性能约束；更新 Profile 统计口径。
- 后端：无 schema/API 变更。
