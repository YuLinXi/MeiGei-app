## Why

`exercise-library-taxonomy-import` 把内置动作做到了 1081 条 + 双轴（部位 × 器械）+ 子分类，但落地后暴露一个**更底层的分类法问题**和两类**体验/数据问题**：

**（底层）分类层级是「漂」的，浏览树与高亮树两套对不齐。** 当前有四套并行的分类轴各做各的活：`category`（部位浏览）、`subcategory`（浏览收窄）、`equipmentType`（器械正交）、`MuscleRegion`（高亮染色）。其中：

- `category`（15 父级）层级不一致——「胸/背/腿」是部位（区域），但「斜方肌/二头/三头/前臂/小腿」直接拿单块肌肉冒充父级。
- `subcategory`（浏览次级）与 `MuscleRegion`（高亮叶级）**几乎每个父级都错位**：胸有「上胸/中下胸」子分类但高亮只有一块 `chest`；肩有「中束」子分类但高亮无 `deltSide` 区；背有「中背」但无对应高亮区；腿有 `adductors` 高亮区但无子分类。浏览和高亮本该是「同一套肌群的两级」，实际是两棵脱节的树。

**（体验）展示扛不住 1000+ 规模。** 分组头失效（`LibraryGroup.classify` 只认 32 个老 code，928 导入条全落「单关节」）；部位轴做成 15 个横滚 chip，大半在屏外，设计原意（训记式左栏竖排）没落地。

**（数据）训记源数据带进噪声。** 「（版本2/3）」近重复、`地板飞鸟（中难度）（负重）` 难度变体、`XX-Tabata/进阶版` 计时命名条目——撑大库容、污染搜索、断裂历史曲线。

本 change 一次解决三层：**确立「①解剖肌群三级树」为唯一分类原则**（终结浏览/高亮对不齐）→ 在此之上重做训记式左栏 + 修分组 → 数据去噪 + 补搜索。

> **与 `exercise-library-taxonomy-import` 的关系**：本 change 是其分类法的**收敛与体验/数据延伸**。依赖 taxonomy-import 先归档（同改 `ExerciseViews.swift` / `ExerciseCategory.swift` / `MuscleRegion.swift` / 内置数据文件）。与 `muscle-map-detailed-art`（高亮美术债）强相关：本 change 把浏览树升到三级，高亮美术按「优雅回退」分批跟进、不阻塞。

## What Changes

### A. 确立分类核心原则：单一解剖三级树（部位 → 肌肉 → 肌头）

- **采用「①解剖肌群」为浏览主干**（业界动作库主流：Hevy/训记/MuscleWiki 同此），器械/动作模式/复合孤立等降为正交筛选标签，不当主干。
- **钉死为三级**，终结层级漂移：
  - **L1 部位**（解剖 8 + 非解剖 3，共 **11**，取代旧 15）：解剖类 `胸/背/肩/手臂/腿/臀/核心/颈部`；非解剖类 `有氧/功能性/热身拉伸`。
  - **L2 肌肉**：插入「肌肉」浏览层（如 胸→胸大肌；背→背阔肌/斜方肌/菱形肌/竖脊肌；手臂→肱二头肌/肱三头肌/前臂；腿→股四头肌/腘绳肌/内收肌/小腿）。
  - **L3 肌头/肌区（按需，不强配）**：仅给用户真会区分的肌肉配（胸大肌→上胸/中下胸；三角肌→前束/中束/后束；腹直肌→上腹/下腹），其余 L3 留空。
- **L1 收缩映射**：`二头/三头/前臂 → 手臂` 的 L2；`小腿 → 腿` 的 L2；`斜方肌 → 背` 的 L2；`颈部` 保持独立 L1。
- **浏览树与高亮树合并为同一棵**：浏览走完整三级；高亮把动作 `primaryRegions/secondaryRegions` 染到「美术支持的最深一级」，**美术不够深则优雅回退到父级**（如美术未拆上/下胸 → 染整块胸）。一个动作的目标肌钉在最深已知层级，浏览归位与高亮染色**同源**。

### B. 动作库版式重做为训记式左栏（路 B）

- 部位轴改**固定窄左栏竖排**（全部 / 我的 / 11 个 L1，带数量角标），有下级的 L1 **就地手风琴展开** L2、L2 再展开 L3 做逐级收窄；**删除横滚 chip 部位/子分类行**。
- 器械轴留右侧顶部 `HorizontalChipPicker`，与解剖树正交。
- **右侧列表分组改用解剖树层级**：删除 `LibraryGroup`（复合推/拉/腿/单关节）。选「全部」→ 按 L1 分段；选 L1 → 按 L2 分段；选 L2 → 按 L3 分段（无下级则平铺）。
- 自定义动作：左栏顶「我的」入口；选具体部位时按 `primaryMuscle` 并进对应段。
- 同段排序：curated（已精修 153）优先。

### C. 搜索补中文模糊

- 在「中文子串」基础上加**多关键词 AND**（按空格拆词、顺序无关；「杠铃 卧推」命中「平板杠铃卧推」）。有 query 时忽略左栏、全库匹配、结果按 L1 分段、左栏选中态淡化；器械 chip 仍叠加。**不做**拼音/英文 code/别名/相关度排序/debounce/索引。

### D. 发布前数据去噪（仅作用于未发布的导入 code，可自由删改、不触发历史合并）

- **删间歇·Tabata 全部 22 条**；`有氧` 仅剩稳态有氧。
- **版本号去重**：有本体的 9 组留本体删版本（−10）；孤儿版本 9 条去尾缀转正改名（含洗 code 后缀）。
- **难度/负重变体合并** 13 族为本体（−18）；负重/难度由记录的重量×次数体现。
- 库容 1081 → ~1031；库内不再有「（版本N）」字样。

## Capabilities

### Modified Capabilities

- `exercise-library`：分类轴由「两级肌群 + 父级内子分类」收敛为**单一解剖三级树（L1 部位 11 / L2 肌肉 / L3 肌头按需）**，浏览与高亮合并同源；`MuscleRegion` 需补 `deltSide`（肩中束）、`rhomboids`（中背）等缺失区；新增「发布前内置动作去噪清理」Requirement。
- `workout-tracking`：「动作库版式」Requirement 改为「训记式左栏（解剖树三级逐级手风琴）× 器械顶部 chip + 按树层级分段 + 中文模糊搜索」。

## Impact

- **iOS 模型层**：`Models/ExerciseCategory.swift`——L1 由 15 收缩到 11（语义改为真部位），新增 L2「肌肉」层与 L1→L2→L3 树定义；`Models/MuscleRegion.swift`——补 `deltSide`/`rhomboids` 等缺失区，与树对齐；`Models/BuiltinExercise.swift`——`category` 改为 L1、`subcategory` 重锚为 L3 肌头、L2 归属由 `primaryRegions` 派生；`BuiltinExercise+Imported.swift`——重映射全部导入条到新三级树 + 去噪（删 Tabata 22/版本 10/难度 18、改名 9）+ 功能性补子分类。
- **iOS 视图层**：`Workout/ExerciseViews.swift`——左栏三级手风琴；删 `LibraryGroup`；右侧按树层级分段；搜索多词 AND + 全局通道；自定义并段；curated 优先。
- **高亮（解耦、不阻塞）**：高亮按「最深可染级 + 优雅回退」渲染；`deltSide`/`rhomboids`/L3 paintable 区块的美术补绘归 `muscle-map-detailed-art` 债，分批跟进。
- **数据安全前提**：删/改 code 全属 taxonomy-import 导入的 928 条，且该批在 build 3（第三次 TestFlight）之后才入库——**未随任何已发布 build 到达用户**，故删改干净、无孤儿历史、无需历史合并。原 153 curated 已发布、冻结，不删改其 code（仅可在树上重新归位 L1/L2/L3，code 不变）。
- **historyKey**：`builtinCode ?? customId ?? name` 计算式不变。
- **同步契约**：`BuiltinExercise` 仍为随包只读静态数据；不改 `WorkoutPlan`/`Workout`/`CustomExercise` 的同步聚合与 LWW/幂等。
- **依赖**：依赖 `exercise-library-taxonomy-import` 先归档；与 `muscle-map-detailed-art` 协同。
- **非影响**：不改后端；不动训练记录/休息计时/Team/计划引用（计划引用 code，归位不改 code）。

## Non-goals

- **不做** L4（肌纤维方向/分段）——三级封顶。
- **不做** L3 强配——仅给用户真会区分的肌肉配肌头，其余留空。
- **不做** 动作模式（推/拉/铰链）或复合/孤立 作为浏览主干——至多作为二期正交筛选标签。
- **不做**「最近用过 / 常用动作」置顶——留二期。
- **不做** 拼音 / 英文 code / 别名 / 搜索相关度排序 / debounce / 索引——搜索只做「中文子串 + 多关键词 AND」。
- **不做** 高亮美术 L3 区块的即时全量补绘——按「优雅回退」分批，归 `muscle-map-detailed-art` 债，不阻塞浏览升级。
- **不做** 已发布老 code（153 curated）的删除/改名——只在树上重新归位、code 不变。
- **不做** 训记写实解剖线稿配图、动图/视频——沿用自绘纸感矢量。
- **不做** 后端动作库下发、动作云同步——内置动作随包只读发布。
- **不做** 自定义动作（`CustomExercise`）的 L1/L2/L3 录入升级——沿用现有行为。
