## ADDED Requirements

### Requirement: 训练组热身切换保持稳定组序

iOS SHALL 将 `WorkoutSet.setIndex` 作为同一动作内训练组的稳定原序。用户在正式组与热身组之间切换时，系统 MUST 只更新有效热身状态，MUST NOT 因该切换修改目标组或其它组的 `setIndex`。

展示时，热身组 MUST 继续排在所有正式组之前；热身段和正式段内部 MUST 分别按稳定 `setIndex` 升序展示。正式组编号 MUST 依据该展示序重新派生。连续切换同一组两次后，所有训练组的稳定原序和最终展示序 MUST 恢复到切换前状态。

对于旧数据中以 `setTypeRaw == "warmup"` 表达热身、但 `isWarmup == false` 的训练组，系统 MUST 以有效热身状态判断切换方向；用户执行一次「改回正式组」后，该组 MUST 成为正式组，并 MUST 将旧 raw 值归一为当前结构类型表示。

#### Scenario: 取消原生热身组后进入正式段原始位置

- **WHEN** 某动作原序依次为 2 个热身组和 3 个正式组，用户取消第 2 个热身组的热身标记
- **THEN** 第 1 个热身组仍排在最前
- **AND** 被取消标记的组按原始 `setIndex` 成为第 1 个正式组，而不是移动到正式段末尾
- **AND** 所有组的 `setIndex` 均保持不变

#### Scenario: 正式组误标后撤销恢复原位

- **WHEN** 某动作有 3 个正式组，用户将第 2 个正式组标为热身后再次取消热身标记
- **THEN** 该组恢复为第 2 个正式组
- **AND** 最终展示顺序与两次切换前一致
- **AND** 两次切换均未修改任何组的 `setIndex`

#### Scenario: 取消热身不改变新增正式组预填来源

- **WHEN** 某动作包含较轻的前置热身组和多个正式组，用户取消其中一个热身组的标记后点击「加一组」
- **THEN** 新正式组仍以稳定原序中最后一个正式组作为重量与次数预填来源
- **AND** 不因取消热身而错误使用前置组的较轻数据

#### Scenario: 旧 raw 热身一次切回正式组

- **WHEN** 某旧训练组满足 `setTypeRaw == "warmup"` 且 `isWarmup == false`，用户执行「改回正式组」
- **THEN** 该组的有效热身状态变为 false
- **AND** `setTypeRaw` 被归一为当前正式结构类型
- **AND** 该组的 `setIndex` 保持不变
