## Why

当前训练组在正式组与热身组之间切换时，会把该组的 `setIndex` 改为当前最大值。取消热身标记因此会把原本位于正式训练前的组移动到正式组末尾，且反复切换无法恢复原顺序，还会改变「加一组」预填和下一组推荐所依赖的顺序语义。

## What Changes

- 将热身切换收敛为纯语义变更：只更新热身状态，不修改组的稳定 `setIndex`。
- 继续保持「热身组整体吸顶、每个分段内按 `setIndex` 排序」的展示规则。
- 正式组临时标为热身后再取消时，恢复到原正式组相对位置；原生热身组取消后按其原始位置进入正式段。
- 兼容旧 `setTypeRaw == "warmup"` 数据，确保一次操作即可正确取消热身。
- 补充顺序可逆、旧数据兼容与正式组预填来源的回归测试。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `workout-tracking`: 调整训练组热身标记切换后的排序要求，使切换不再改变稳定组序，并明确旧热身 raw 值的取消行为。

## Impact

- iOS 领域模型：`WorkoutExercise.toggleWarmup(_:)`。
- iOS 展示及交互：继续复用现有 `displaySortedSets`、正式组编号、键盘跳转和休息策略，无新增 UI。
- iOS 测试：增加热身切换排序、可逆性、旧 raw 值及预填来源覆盖。
- 后端、数据库、同步 DTO 与 API 契约均不变；既有 `setIndex` 和 `isWarmup` 字段继续原样同步。

## Non-goals

- 不新增训练组拖拽排序或取消标记后的位置选择器。
- 不按重量、完成状态或其它训练数据自动排序。
- 不迁移或猜测已经被旧逻辑改写且无法还原的历史 `setIndex`。
- 不改变热身组不参与训练量、PR、历史曲线及正式组统计的现有口径。
- 不改变超级组整轮热身或递减组整体热身规则。
