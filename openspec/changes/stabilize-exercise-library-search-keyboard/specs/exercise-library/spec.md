## ADDED Requirements

### Requirement: 动作库顶部搜索框键盘稳定性

动作库 SHALL 保留当前自定义顶部搜索框视觉，并在系统键盘弹出、收起、Tab 切换返回和添加动作抽屉展示期间保持搜索框顶部位置稳定。搜索框 MUST 位于状态栏、Dynamic Island、刘海和「返回上个 App」提示区域下方，MUST NOT 因键盘避让上移到系统状态区域内。

动作库 Tab 与添加动作抽屉 SHALL 复用同一套搜索框布局和键盘稳定策略；搜索查询、部位筛选、器械筛选和动作列表过滤 MUST 保持现有语义。

动作库 Tab 的主体内容 SHOULD 填充到系统 TabBar 上方的可用区域；右下悬浮添加按钮 MUST NOT 通过底部安全区插入额外抬高主体内容，造成列表底部与 TabBar 之间出现异常大留白。

#### Scenario: 动作库 Tab 搜索不进入状态栏

- **WHEN** 用户进入动作库 Tab 并点击顶部搜索框使系统键盘弹出
- **THEN** 搜索框位置保持在顶部安全区下方，不与状态栏、Dynamic Island 或返回上个 App 文案重叠

#### Scenario: Tab 切换返回后搜索仍稳定

- **WHEN** 用户从动作库 Tab 切到其它 Tab 后再返回动作库 Tab，并点击顶部搜索框使系统键盘弹出
- **THEN** 搜索框位置与首次进入动作库时一致，不出现额外顶部留白或向上重叠

#### Scenario: 添加动作抽屉搜索体验一致

- **WHEN** 用户在训练进行中打开添加动作抽屉并点击顶部搜索框使系统键盘弹出
- **THEN** 抽屉内搜索框保持在顶部安全区下方，键盘弹出时不向状态栏区域移动

#### Scenario: 搜索过滤语义不变

- **WHEN** 用户在稳定后的搜索框输入关键词并切换部位或器械筛选
- **THEN** 动作列表仍按现有搜索、部位和器械组合规则过滤，左侧分类和右侧器械索引仍可正常使用

#### Scenario: 动作库底部内容不被悬浮按钮抬高

- **WHEN** 用户进入动作库 Tab，列表内容多于一屏且右下角显示添加自定义动作按钮
- **THEN** 动作库主体内容继续填充到系统 TabBar 上方，不因悬浮按钮预留整块底部 inset 而提前截断
