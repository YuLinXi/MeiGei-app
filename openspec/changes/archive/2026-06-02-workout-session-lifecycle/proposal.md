## Why

训练记录是 MeiGei 最核心的模块，但当前实现缺少一个明确的「训练会话生命周期」，导致一系列功能 bug 与产品语义矛盾（均已在代码中定位）：

- **假暂停**：进行中页的「暂停」按钮（`WorkoutViews.swift:539`）仅切换图标，`paused` 变量从不参与计时；REC 计时是纯墙钟 `now − startedAt`（`:531`），暂停对计时与记录的时长零影响。
- **结束不冻结**：`TimelineView` 计时位于 `if !isEnded` 判断之外，训练结束后秒数仍持续增长，未冻结为 `endedAt − startedAt`。
- **结束态语义矛盾**：结束后暂停/结束按钮消失（`:538`），无任何「恢复」入口，会话被单向焊死；但「添加动作」与每个 `SetRow` 均未加结束守卫，用户仍可加动作、改重量、勾完成——且这些改动不会重算 PR、不会更新已发出的 Team 打卡，导致本地与 Team 数据静默不一致。
- **可多开活跃会话**：`startBlank()`（`:242`）无条件 `insert` 新 `Workout`，从不检查是否已有未结束会话，可同时存在多个 `endedAt == nil` 的训练；`addToTodayWorkout()` 用 `.first(where: endedAt==nil)` 归属动作时行为不确定。
- **无法删除训练记录**：全 App 内 `Workout` 聚合根没有任何删除入口（列表无 swipe、详情菜单无删除、日历无 swipe），误触产生的「0 动作」鬼影记录永久污染最近列表与统计，且无法清除。
- **进行中训练混入历史**：未结束会话直接出现在「最近训练」常规行中，无视觉区分、无「继续训练」语义。

这些散点的根因是同一个：**一次训练会话从未被定义为一个有限状态机**。本 change 引入参照成熟严肃训练工具（Strong / Hevy）收敛出的会话生命周期，一次性消除上述矛盾。

## What Changes

- **会话状态机**：将训练会话明确为三态——`进行中（ACTIVE，唯一）` / `已完成（FINISHED）`，以及隐含的「无会话」。所有「开始训练」入口统一经一个守卫，保证同一时刻至多一个进行中会话。
- **移除会话级暂停**：删除进行中页的暂停按钮。会话总时长为墙钟（含组间休息，符合品类标准）；「歇一下」的真实需求由既有的组间休息计时器（`RestTimerController`）承载。
- **结束二次确认**：「结束」改为弹确认（展示「X 动作 · Y 组完成」摘要），取消则回到进行中，避免误触归档。
- **结束后冻结计时**：已完成训练的计时显示冻结为 `endedAt − startedAt`，不再持续增长。
- **已完成 = 只读摘要 + 显式编辑**：进行中与已完成使用不同呈现。已完成态默认只读；通过显式「编辑」进入编辑，编辑期间方可加/删动作、改组；保存后 MUST 重算 PR 并更新对应 Team 打卡。不做「恢复到进行中」。
- **继续 / 丢弃进行中会话**：训练首页顶部以醒目「继续训练」横幅呈现唯一进行中会话；尝试开始新训练而存在进行中会话时，提示「继续 / 丢弃」。「丢弃」删除该进行中会话。
- **删除已完成训练**：已完成训练记录支持删除（列表左滑），走软删墓碑同步至后端。

## Capabilities

### Modified Capabilities
- `workout-tracking`: 新增「训练会话生命周期」相关行为规约（单一活跃会话、墙钟计时与移除暂停、结束确认与冻结、只读摘要与编辑闭环、继续/丢弃、删除训练记录）。

## Impact

- **iOS 端（主要）**：
  - `Workout` 模型：新增派生的会话状态判定（基于既有 `endedAt`），**不新增暂停相关持久化字段**。
  - `WorkoutViews.swift`：移除暂停按钮；拆分进行中（Live 记录）与已完成（只读摘要 + 编辑）两套呈现；结束确认弹窗；结束后冻结计时；编辑保存后重算 PR / 更新打卡。
  - 训练首页：新增「继续训练」横幅；进行中会话从常规历史行中分离；最近/历史列表新增左滑删除。
  - 统一所有「开始训练」入口（`startBlank` / `start(from:)` / `PlanDetailView.startWorkout` / `ExerciseDetailView.addToTodayWorkout`）经单一活跃会话守卫。
- **后端**：核对 `Workout` 聚合根软删墓碑 push 路径（CLAUDE.md 已知坑：`updateById` 不写 `@TableLogic` 字段，需显式 `softDelete` 且连带删子树）；如缺失需补齐。
- **数据迁移**：无 schema 变更（不新增字段）。

### Non-goals（本 change 明确不做）
- 不做会话级真暂停 / 「有效训练时长（剔除休息）」指标，不为此新增 `pausedAt` / `accumulatedActiveSeconds` 字段。
- 不做「将已完成训练恢复为进行中」（用编辑覆盖该需求）。
- 不做计划模块的动作项删除/重排、动作库假搜索框、HealthKit 真实授权态、Team 成员昵称等其它模块的体验缺口（另立 change）。
- 不改动既有训练首页视觉基线、休息计时器圆环、PR 配色等 `add-neon-design-system` 已定的视觉规约。
