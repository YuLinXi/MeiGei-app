# 设计：动作库体验升级

> 本文记录关键设计决策与取舍。背景见 `proposal.md`，验收步骤见 `tasks.md`。

## 依赖与排序

本 change 与 `exercise-library-taxonomy-import` **同改** `ExerciseViews.swift` / `ExerciseCategory.swift` / `MuscleRegion.swift` / 内置数据文件，MUST 在其**完成并归档后**实施。与 `muscle-map-detailed-art`（高亮美术债）协同——本 change 把浏览树升三级，美术按 D8「优雅回退」分批跟进、不互相阻塞。

## D0：分类核心原则——单一解剖三级树（本 change 的地基）

调研业界公认的 5 套分类轴后定调：

| 轴 | 本质 | 角色 |
|---|---|---|
| ① 解剖肌群 | 练哪块肌肉 | **浏览主干**（Hevy/训记/MuscleWiki 同此，C 端心智「今天练胸」） |
| ② 动作模式（推/拉/铰链） | 怎么发力 | 至多二期正交标签（排计划向，C 端不按此找动作） |
| ③ 力学（复合/孤立·推/拉） | 关节与力性质 | 至多正交筛选标签（ExRx 做法） |
| ④ 器械 | 用什么练 | 永久正交副轴 |
| ⑤ 肌肉参与角色（主动/协同/稳定） | 每块肌肉职责 | 挂在动作上的标签（即 primary/secondary regions） |

**决策**：① 为唯一浏览主干，钉死三级，其余降为正交标签/动作级标签。三级解剖深度是解剖学标准答案（部位→肌肉→肌头/肌区），封顶不做 L4 纤维方向。

---

## D1：三级树定稿（含 L1 收缩映射）

```
L1 部位       L2 肌肉              L3 肌头/肌区（按需，不强配）
─────────────────────────────────────────────────────────────
胸           胸大肌               上胸(锁骨头) · 中下胸(胸肋头)
背           背阔肌               —
            斜方肌               (上斜方 · 中下斜方，可选)
            菱形肌(中背)          —
            竖脊肌(下背)          —
肩           三角肌               前束 · 中束 · 后束
手臂         肱二头肌             —
            肱三头肌             (长头 · 外侧头 · 内侧头，可选)
            前臂                 —
腿           股四头肌             —
            腘绳肌               —
            内收肌               —
            小腿                 —
臀           臀大肌 · 臀中肌       —
核心         腹直肌               上腹 · 下腹
            腹斜肌               —
颈部         颈部肌               —
──── 非解剖并行枝（无 L2/L3，无高亮） ────
有氧 / 功能性 / 热身拉伸
```

**用户定稿的 4 个收尾点**：
1. **小腿** 并入「腿」做 L2（不独立 L1）。
2. **颈部** 保持独立 L1。
3. **二头/三头/前臂** 作为「手臂」的 L2（手臂为 L1，非健美式拆二头/三头当 L1）。
4. **L3 按需不强配**——仅 上胸/中下胸、三角肌三束、上腹/下腹 等用户真会区分处配 L3；其余 L2 的 L3 留空。

**L1 由 15 收缩到 11 的迁移**：`二头/三头/前臂 → 手臂`、`小腿 → 腿`、`斜方肌 → 背`（均降为 L2）；`颈部` 独立保留；非解剖 `有氧/功能性/热身拉伸` 保留为并行枝。

## D2：浏览树与高亮树合并同源 + 优雅回退（终结对不齐）

**问题**：旧 `subcategory`（浏览）与 `MuscleRegion`（高亮）是两棵脱节的树，几乎每个父级都错位。

**决策**：**只有一棵解剖树**，浏览与高亮共用。

- 每个动作把目标肌钉在**最深已知层级**（如「上胸」），用 `primaryRegions/secondaryRegions` 承载。
- **浏览归位**：L1=`category`（显式存）；L2= 由 `primaryRegions` 派生（命中哪块肌肉）；L3= `subcategory` 重锚为「肌头」（显式存，按需）。浏览据此走完整三级。
- **高亮染色**：把目标区染到**美术支持的最深一级**；美术不够深则**回退到父级**（美术未拆上/下胸 → 染整块胸；三头未拆头 → 染整三头）。
- → 浏览归位与高亮染色**同源于 `primaryRegions`+`subcategory`**，不再各长一套，结构性消除错位。

**MuscleRegion 需补缺失区**（与树对齐）：新增 `deltSide`（肩中束）、`rhomboids`（中背/菱形）等当前缺失的 L2/L3 paintable 节点。新增区块的美术补绘归 `muscle-map-detailed-art` 债；补绘前这些节点按「优雅回退」染最近可染父级或暂不高亮，**不阻塞**浏览升级。

**数据债衔接**：928 导入条多数 `primaryRegions` 为空（region 回填是既有数据债）。回填前这些动作有 L1、无 L2/L3，落「L1 → 全部」，路 B 仍可用；回填后自动归到 L2/L3。

## D3：动作库版式重做·路 B 左栏（三级手风琴）

| 维度 | 横滚 chip（现状） | 左栏竖排（本 change） |
|---|---|---|
| 可见性 | 11 部位大半在屏外 | 一屏纵览 |
| 首屏占用 | 器械+部位+子分类三条横滚 | 仅左栏 + 顶部一条器械 chip |
| 层级 | 又一条横滚 chip | L1→L2→L3 就地手风琴，层级直观 |

**决策**：部位轴改固定窄左栏（约 88–96pt，自身可滚），竖列「全部 / 我的 / 11 个 L1」+ 数量角标；有下级的节点**逐级手风琴展开**（L1 展开 L2，L2 展开 L3），点哪级就过滤到哪级；器械轴留右侧顶部 chip；**删除独立子分类横滚 chip 行**。

**展示层折叠规则**：数据树仍保留完整 `L1→L2→L3`，但若某 L1 只有一个 L2 且该 L2 有 L3，UI SHALL 折叠这个唯一 L2，直接展示 L3。典型例子：`胸→胸大肌→上胸/中下胸` 在左栏显示为 `胸→上胸/中下胸`；`肩→三角肌→前束/中束/后束` 显示为 `肩→前束/中束/后束`。内部筛选、归位、高亮仍使用完整 L2/L3，不删除 `胸大肌` 或 `三角肌` 节点。

**为何路 B（就地下钻）而非靠滚**：背/核心/功能性等大桶，靠「右侧分段 + 滚动」救不了；三级下钻能从 100+ 收窄到 10~30 条。

## D4：右侧分组改用解剖树层级（修复分组失效）

**问题**：`LibraryGroup.classify(code)` 仅认 32 个老 code → 928 导入全落「单关节」，分组头失真。

**决策**：**删除 `LibraryGroup` 整套**。右侧分段由解剖树驱动：

```
选「全部」（无搜索）        → 按 L1 部位分段
选某 L1（如「胸」）         → 按 L2 肌肉分段（胸大肌…）
选某 L2（如「胸大肌」）     → 按 L3 肌头分段（上胸/中下胸）；无 L3 则平铺
选到底/无下级              → 平铺
搜索态（有 query）         → 见 D6，按 L1 分段
```

若命中 D3 的展示层折叠规则，选某 L1 后右侧直接按 L3 分段：例如选「胸」直接分「上胸/中下胸」，选「肩」直接分「前束/中束/后束」。

## D5：搜索＝极简中文模糊（多词 AND）

**决策**：只做两件事——① 中文子串（`localizedCaseInsensitiveContains`）；② **多关键词 AND**（query 空格拆词，每词都需被名称包含，顺序无关）。

**明确不做**：拼音、英文 code、别名、相关度排序、debounce、预建索引（用户要极简；1031 条单次按键亚毫秒，无须索引/debounce）。

**全局通道**：有 query 时忽略左栏（L1/L2/L3）过滤、全库匹配、结果按 L1 分段、左栏选中态淡化；**器械 chip 仍叠加**（器械是正交硬过滤）。清空 query 回到左栏当前选择浏览态。

## D6：发布前数据去噪——为何可「自由删改、不走历史合并」

**安全前提（git 证据）**：

```
7449a5b  build 号 bump 2→3，准备第三次 TestFlight 发版   ← build 3 在此切
6428456  全量导入 928 动作                                ← 导入在 build 3 之后
```

→ 928 条导入动作（含所有 `_V2/_V3`、Tabata、难度变体）**未随任何已发布 build 到达用户**，「code 发布即冻结」对这批尚未触发 → **干净删除/改名，无孤儿、不需历史合并**。原 153 curated 已发布、冻结，本 change **不删改其 code**（仅可在树上重新归位 L1/L2/L3）。

| 类别 | 规则 | 量 |
|---|---|---|
| 间歇·Tabata | 全删（按计时方案命名，非动作本体） | −22 |
| 版本号·有本体 | 留无版本本体，删 `_V2/_V3` | −10（9 组） |
| 版本号·孤儿 | 去名称尾缀 + 洗 code 后缀「转正」（不删） | 改名 9 |
| 难度/负重变体 | 合并为一条本体（负重/难度由记录重量×次数体现） | −18（13 族） |

**幸存者规则**：优先留无版本/无难度后缀、code 最干净的本体；已回填 regionData 者优先（实测这批均未回填）。

## D7：功能性补子分类（非解剖枝的 L2）

`功能性` 属非解剖并行枝、无高亮，但路 B 下点它右侧 102 条平铺难下钻。**决策**：在功能性枝下补 5 个浏览子类（覆盖 102 条采样）：

| 子类 | 典型动作 |
|---|---|
| 爆发奥举 | 高翻 / 抓举 / 挺举 / 火箭推 / 波比跳 / 土耳其起立 |
| 抗旋稳定 | Pallof 推 / 巫师扭转 / 9090 旋转 |
| 髋臀激活 | 弹力带蚌式 / 螃蟹行走 / 各类外展 |
| 负重搬运 | 农夫行走 / 壶铃甩 / 哑铃摆荡 |
| 动态控制 | 熊爬 / 下犬式 / 动态蝴蝶式 / 剪刀踢腿 |

逐条归类是数据债、可分批；未归类落「全部」、不阻塞。命名避开「核心」的「核心稳定」，用「抗旋稳定」。

## 风险与回归边界

- **historyKey 不变**：计算式不动；删/改 code 未发布、无历史。L1/L2/L3 归位不改 code，故历史曲线/PR 不断裂。
- **计划引用**：`WorkoutPlan.items` 引用 code，归位不改 code，安全；被删 code 未发布、无线上计划引用。
- **高亮解耦**：浏览升三级不等高亮升三级；高亮按「最深可染级 + 回退」，新区美术分批补，缺区不崩（回退父级或暂隐）。
- **编译兜底**：L1 由 15→11、删 `LibraryGroup`、`MuscleRegion` 增 case 后，靠 `xcodebuild` + 单测找全引用点与非法枚举值。
- **迁移工作量**：重映射全部 ~1031 条到三级树是体力活，按 L1 分批；未映射条目落「L1→全部」保持可用。
