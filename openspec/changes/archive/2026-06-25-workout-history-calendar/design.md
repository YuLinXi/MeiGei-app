# Design — 历史训练日历

## UI 形态

采用训练页入口 + 月历 + 底部当日抽屉：

```text
训练首页
  └─ 日历按钮
      └─ History Calendar
          ├─ 月份标题 / 独立月份选择按钮 / 左右切月 / 今天
          ├─ 6x7 月历网格
          ├─ 紧凑本月摘要
          └─ 选中日期底部抽屉
              └─ 当日训练卡片 → WorkoutDetailView
```

月历负责扫视训练节奏；当日抽屉负责快速进入训练日志。历史日历主界面采用一屏布局，不使用页面级滚动，也不让底部抽屉覆盖日历网格。V1 不做搜索、年度热力图、PR 榜单或复杂图表，避免重新引入历史大页的渲染压力。

「今天」是月份控制的一部分，不放在导航栏右上角；导航栏只承担返回和页面标题。当前已显示今天所在月份且选中今天时，隐藏「今天」入口，避免无意义常驻按钮。

主月历日期格采用纸感摘要徽条，不复制参考 App 的蓝/黑记录块。每个有训练的日期展示：

- 日期数字。
- 当天训练量短文本，例如 `3.6t`。
- 当天第一条训练标题；同日多次训练时追加 `+N`。

月份标题旁提供独立圆形日历按钮，点击打开月份档案 sheet，月份文字本身不承担点击职责。月份档案按年份分组倒序展示，从当前月到最早训练月；每个年份有明确表头，每行展示月份、训练天数胶囊、31 个小方点组成的日期密度条、训练次数和进入箭头。右侧提供年份索引，顶部箭头用于快速回到列表顶部。选择某月后直接跳转到该月；本月选中今天，非本月优先选中该月第一天有训练的日期，否则选中 1 日。

## 数据设计

`WorkoutHistoryStore` 在现有 projection 构建时顺带生成日历摘要：

- `CalendarDaySummary`：日期、训练 row 列表、训练量、正式组数、PR 标记。
- `CalendarMonthSnapshot`：月份、42 个日期格、月度聚合（训练次数、训练量、正式组数）。
- `CalendarMonthArchiveItem`：月份、训练天数、训练次数、训练量、正式组数与有训练的日号集合，用于月份档案列表。
- `CalendarYearArchiveGroup`：年份与该年份下月份档案项，用于年份分组和右侧年份索引。
- View 通过按月方法读取纯值快照，不直接遍历 `Workout.exercises.sets`。

「最长连续」不再作为产品指标展示或计算；Profile projection 仅保留总训练次数。
「平均时长」不再作为产品指标展示或计算；单次训练行可继续展示该训练自身的持续时间。

## 性能策略

- 月份切换只做 42 个日期格的 dictionary lookup。
- 点训练卡片时才按 `workoutId` fetch 单条 `Workout` 并进入详情。
- 导入大量历史后，projection 可重建；日历 UI 显示上一次完成的快照，不在 `body` 内触发全量重算。
- 本次不新增服务端 summary。若未来 10000+ 训练或冷启动瓶颈明显，再评估后端按月摘要接口。
