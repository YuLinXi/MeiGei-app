# Team 拍一拍本地验收数据

## 启动与重置

在 `backend/` 目录启动本地后端：

```bash
APP_DEV_TOKEN=true ./gradlew bootRun
```

模拟器使用本地环境并通过开发账号登录时，`POST /auth/dev/token` 会自动创建或重置验收数据。需要重新回到初始状态时，退出后重新开发登录即可，也可以手动调用：

```bash
curl -X POST http://localhost:8001/auth/dev/token
```

重置只清理“拍一拍验收 Team”中会被手工操作改变的当日 checkin、当前账号发出的当日 nudge 和接收偏好，不影响原“本地测试 Team”。

## 验收 Team 基线

进入“拍一拍验收 Team”，顶部应显示 `2 / 6 今日已练`。点击今日进度 pill 或整个头像栈打开成员 sheet。sheet 不展示“拍一拍队友”标题、Team 名小文案或接收开关；展示 4 名允许接收拍一拍的队友，不展示当前账号和已关闭接收的成员。顶部仅保留系统 drag indicator，通过下滑手势关闭，不显示右上角关闭按钮。

| 成员 | 初始状态 | 验证点 |
| --- | --- | --- |
| 测试队友 | 今日已分享 | 显示“已分享”，不能拍一拍 |
| 测试成员阿凯 | 今日已分享 | 显示“已分享”，不能拍一拍 |
| 老周 | 已预置当日 nudge | 初始直接显示“已拍”，不能重复发送 |
| 小满 | 对方关闭该 Team 接收偏好 | 不出现在“拍一拍队友” sheet 中 |
| 阿岳 | 可拍 | 用于验证成功、触觉和按钮状态 |

Team 详情页在 header 卡下方相邻展示“训练完成后自动分享”和“接收这个 Team 的拍一拍”两个设置卡，当前账号均默认为开启。切换后应保存，失败时恢复旧值并提示；重新开发登录会恢复为开启。

## 自动化覆盖

发送者每日 5 人上限、接收者每日 3 条 APNs 上限、并发配额锁、幂等与推送抑制不写入手工账号基线，避免阻断正常成功路径。运行以下测试覆盖这些边界：

```bash
./gradlew test --tests com.dontlift.team.TeamNudgeServiceTest
```

本地未配置 APNs 凭据时推送会按设计降级为 no-op；标题、正文和 `type=team_nudge` / `teamId` payload 由上述测试验证，真实通知展示与点击仍需配置 APNs 后用真机验证。
