## 字段说明

- `序号`：动作在当前审核清单里的顺序。
- `来源`：`curated` 是原 153 条精选动作；`imported` 是后来导入的长尾动作。
- `动作编码`：内置动作 code，发布后尽量不改。
- `动作名称`：App 内展示的中文动作名。
- `器械`：当前器械分类。
- `原始一级分类 / 原始子分类`：Swift 数据里的原始字面量。
- `实际一级分类 / 实际二级肌肉 / 实际三级肌区`：App 归一化后的实际分类。
- `用户可见路径`：用户在动作库看到的路径，已应用胸/肩单一 L2 折叠规则。
- `主动肌区 / 协同肌区`：当前高亮图使用的肌区。
- `可疑提示`：脚本按名称关键词标出的可疑点，只作为优先复核提示。
- `审核结论`：建议填 `通过`、`调整分类`、`删除`、`合并`、`改名`、`待确认`。
- `审核备注`：人工审核说明，例如“后束飞鸟应归肩后束”“与 XXX 合并”。

## Excel 审核方式

- 推荐直接编辑 `动作分类审核表.xlsx`。
- 黄色列是可直接修改的原列：`器械`、`实际一级分类`、`实际二级肌肉`、`实际三级肌区`、`主动肌区`、`协同肌区`、`审核结论`、`审核备注`。
- `主动肌区`、`协同肌区` 使用中文显示并带下拉。多个肌区可手动用顿号分隔，例如 `三角肌前束、肱三头肌`。
- `平台支持选项` 工作表列出了所有下拉可选项：`审核结论`、`一级分类`、`二级肌肉`、`三级肌区/浏览子类`、`器械`、`肌区`。
- 审完后你只需要告诉我审核完成或补充 `审核结论 / 审核备注` 口径，我会读取你直接调整后的原列并回填到 Swift 动作数据。
