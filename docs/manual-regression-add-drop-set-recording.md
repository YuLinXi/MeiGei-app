# 递减组录入与计划闭环手动回归用例

## 训练录入

1. 在任意动作卡点击底部「递减组」。
   - 期望：动作末尾新增一组，标识显示「递减组」。
   - 期望：默认出现 2 个分段；若前面已有正式组，第 1 段继承上一正式组重量和次数。
   - 期望：新增后数字键盘聚焦到递减组首段重量。

2. 在递减组内录入 `50kg×10、60kg×8、55kg×6`。
   - 期望：App 不提示方向错误。
   - 期望：名称仍显示「递减组」，不出现「递增组」或「混合组」。

3. 点击递减组内「添加一段」，再删除其中一段。
   - 期望：新增段可输入重量和次数。
   - 期望：删除后剩余分段重新按顺序展示，不影响父级组编号。

4. 使用数字键盘的上一项/下一项。
   - 期望：焦点按展示顺序走 `第1段重量 -> 第1段次数 -> 第2段重量 -> 第2段次数`。
   - 期望：在 segment 焦点时点击键盘「加一组」，新增的是普通正式组。

## 组类型切换

1. 在普通正式组右侧更多操作选择「改为递减组」。
   - 期望：原重量/次数成为第 1 段。
   - 期望：系统追加 1 个空白分段。

2. 在递减组右侧更多操作选择「改回普通组」。
   - 期望：先出现确认弹窗。
   - 期望：确认后使用第一个有效分段回填普通组重量/次数，其它分段被移除。

3. 在热身组右侧更多操作查看菜单。
   - 期望：不直接展示「改为递减组」。
   - 期望：先改回正式组后，才可切换为递减组。

## 完成训练与统计

1. 完成一个递减组 `80kg×8 / 60kg×6 / 45kg×8`。
   - 期望：已完成组数增加 1。
   - 期望：训练量增加 `80*8 + 60*6 + 45*8`。
   - 期望：总次数增加 `8 + 6 + 8`。

2. 录入递减组但不勾选完成，结束训练。
   - 期望：该递减组被清理，不进入训练详情。
   - 期望：不影响训练量、PR、历史曲线或计划回写。

3. 完成递减组但保留一个完全空白分段，结束训练。
   - 期望：空白分段被清理或不参与统计。
   - 期望：有效分段仍显示在训练详情中。

4. 递减组最大重量超过历史 PR。
   - 期望：完成训练后出现 PR 识别。
   - 期望：首页动作 PR、训练详情 PR 标记、分享海报 PR 文案一致。

## 训练详情与海报

1. 打开包含递减组的已完成训练详情。
   - 期望：对应组显示「递减组」标识。
   - 期望：详情能看到每个有效分段的重量和次数。

2. 生成分享海报。
   - 期望：总容量按所有有效分段计算。
   - 期望：动作高光行使用最大重量分段作为顶组，并提示额外分段。

## 同步与兼容

1. 设备 A 记录并同步包含递减组的训练，设备 B 下拉同步。
   - 期望：设备 B 能看到同一父级递减组和所有分段。
   - 期望：父级组编号、完成状态、休息记录和备注保持一致。

2. 读取旧训练记录。
   - 期望：缺少 `segments` 字段时按普通组展示。
   - 期望：旧普通组重量/次数仍能参与统计。

3. 新旧客户端交叉编辑同一训练。
   - 期望：冲突仍按 Workout 聚合根 last-write-wins。
   - 风险确认：旧客户端上传同一 workout 可能丢失 segments，新版本应至少保留父级摘要。

## 计划处方

1. 将包含递减组的无计划已完成训练保存为计划模板。
   - 期望：计划项保留递减组 `setPrescriptions`。
   - 期望：计划列表摘要仍显示组数、次数和顶段重量。

2. 从严格计划开始训练，计划项包含递减组处方。
   - 期望：开始训练后生成递减组父级 set。
   - 期望：所有分段按处方落值，父级 `completed=false`。

3. 从自适应计划开始训练，历史中有上次完成递减组。
   - 期望：优先使用上次完成的递减组结构和分段落值。
   - 期望：无历史时才回退到计划处方；再无处方时回退旧 `suggested*`。

4. 完成自适应计划训练并包含递减组。
   - 期望：计划回写后 `suggestedSets` 按父级逻辑组数只增不减。
   - 期望：`suggestedWeightKg/suggestedReps` 使用最大重量分段。
   - 期望：`setPrescriptions` 保留完整递减组结构。
   - 期望：完成页回写回执可撤销，撤销后计划恢复到训练前。

## Team 分享与 Fork

1. 分享包含递减组处方的计划到 Team。
   - 期望：Team 分享版本保留递减组结构、分段数量和次数。
   - 期望：顶层 `suggestedWeightKg`、普通处方重量和所有 segment 重量均为空。

2. 成员 Fork 含递减组的 Team 计划。
   - 期望：新计划模式为自适应。
   - 期望：递减组结构和次数保留。
   - 期望：所有重量为空，且新计划默认进入「未分组」。

3. 自动或手动分享包含递减组的训练打卡。
   - 期望：Team 打卡摘要总组数按父级递减组计 1。
   - 期望：容量按有效分段展开计算。
   - 期望：打卡详情能看到递减组和每个分段。

## VoiceOver

1. 在训练录入页用 VoiceOver 聚焦递减组父级行。
   - 期望：能朗读「递减组」和第几组。

2. 逐个聚焦递减组 segment 输入框。
   - 期望：能区分「第 N 段重量」与「第 N 段次数」。

3. 打开训练详情或 Team 打卡详情。
   - 期望：递减组分段明细可被完整朗读。
